Blog 5: Javascript Error Handling

For my fifth blog post, I will be looking into error handling in Javascript. Just like many programming languages, Javascript allows its developers to make use of error handling. Normally, if there was an error in a script, the script would just stop and print the error to the console. By using a try catch block, Javascript allows us to handle the error and do something other than print it to the console. This is the basic syntax for using a try catch block: 

      
        try {

          // code...

        } catch (err) {

          // error handling

        }

In this case, the code that is in the try block would get executed. If there were no errors, the catch block would be ignored. If there were errors, the control flow would go to the beginning of the catch block. The try catch block will only work for runtime errors. The catch block takes in a variable, in this case err, that is an error object. This error object contains 3 main properties that give information on what went wrong. The first property is the name property that contains the error name. The second property is the message property that contains textual information on the error. The third property is the stack property which contains information on the sequence of calls that lead to the error. Here is an example of using the err properties: 

      
        try {
          causeError; // error, variable is not defined!
        } catch(err) {
          console.log(err.name); // ReferenceError
          console.log(err.message); // causeError is not defined
          // ReferenceError: causeError is not defined at (...call stack)
          console.log(err.stack);

          // Can also show an error as a whole
          // The error is converted to string as "name: message"
          console.log(err); // ReferenceError: lalala is not defined
        }
